# 网络编程

服务端开发的本质是处理网络 I/O：接收客户端请求、解析协议、处理业务、返回响应。理解网络编程的底层原理，是构建高性能服务的基础。

## 网络分层模型

### OSI 七层模型

应用层、表示层、会话层、传输层、网络层、数据链路层、物理层。理论完备但工程实践复杂。

### TCP/IP 四层模型

应用层（HTTP、DNS）、传输层（TCP、UDP）、网络层（IP、ICMP）、网络接口层（Ethernet、Wi-Fi）。互联网的实际标准。

## Socket 编程基础

Socket 是网络编程的抽象，屏蔽了底层协议细节。一个 Socket 连接由五元组标识：源 IP、源端口、目的 IP、目的端口、协议。

### Socket 类型

流式套接字（SOCK_STREAM，基于 TCP）提供可靠、面向连接的字节流服务。数据报套接字（SOCK_DGRAM，基于 UDP）提供不可靠、无连接的数据报服务。

### TCP 连接建立

三次握手：客户端发送 SYN，服务端回复 SYN+ACK，客户端回复 ACK。握手过程携带双方的初始序列号（ISN）。

### TCP 连接终止

四次挥手：主动方发送 FIN，被动方回复 ACK，被动方发送 FIN，主动方回复 ACK。TIME_WAIT 状态持续 2MSL，确保最后的 ACK 能够到达。

## I/O 模型

### 阻塞 I/O

应用程序调用 recvfrom 后，进程阻塞直到数据准备好并拷贝到用户空间。简单直观，但一个线程只能处理一个连接。

### 非阻塞 I/O

应用程序调用 recvfrom 后，立即返回，不管数据是否准备好。进程需要不断轮询，浪费 CPU。

### I/O 多路复用

select/poll/epoll 等系统调用可以同时监控多个文件描述符，任一描述符就绪时返回。进程可以在阻塞等待多个事件，一个线程处理多个连接。

### 异步 I/O

应用程序调用 aio_read 后立即返回，数据准备好并拷贝到用户空间后，内核通知应用程序。真正的异步，但支持有限。

## 网络协议

### TCP 协议细节

TCP 提供可靠传输：序列号保证数据有序，确认应答保证数据到达，超时重传保证数据完整性，流量控制（滑动窗口），拥塞控制（慢启动、拥塞避免）。

### HTTP 协议

HTTP 是应用层协议，基于请求-响应模式。HTTP/1.0 每个请求建立新连接，HTTP/1.1 引入 keep-alive，HTTP/2 引入多路复用，HTTP/3 基于 QUIC。

### WebSocket

WebSocket 是全双工通信协议，客户端和服务端可以随时发送数据。握手基于 HTTP，升级后使用 TCP 长连接。
