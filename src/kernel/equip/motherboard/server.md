# 服务器主板
企业级服务器主板服务于数据中心、云计算和高性能计算场景，与消费级主板的设计思路有本质区别。服务器主板采用 SoC（System-on-Chip）架构，CPU 直接集成内存控制器、PCIe 根复合体、SATA/USB 控制器和安全协处理器，不再需要传统的南桥芯片组。一个低速的外置 BMC（Baseboard Management Controller，基板管理控制器）芯片负责带外管理（IPMI/Redfish）、远程 KVM、虚拟介质挂载等运维功能。

企业级平台的核心差异体现在三个方面：内存通道数量（决定总带宽）、PCIe 通道数量（决定扩展能力）、以及是否支持多路互联（决定单节点的算力上限）。以 Genoa 平台为例，12 通道 DDR5 提供 460 GB/s 的理论内存带宽，128 条 PCIe 5.0 通道可同时驱动 8 块 GPU 和 24 块 NVMe 硬盘——这些都是消费级平台无法企及的扩展规模。

## 与消费级主板的核心差异
服务器主板和消费级主板在多个维度上都有根本性的不同，理解这些差异有助于判断何时应该选择服务器平台。

### RAS 特性
RAS（Reliability, Availability, Serviceability）是服务器主板的灵魂。可靠性方面，服务器主板全链路支持 ECC 内存（从 DRAM 颗粒到内存控制器），能纠正单比特错误、检测双比特错误；PCIe 通道支持 Advanced Error Reporting（AER），可隔离故障链路。可用性方面，热插拔（Hot-plug）覆盖 CPU、内存、PCIe 设备和 NVMe 硬盘——故障部件可以在不停机的情况下被替换。可维护性方面，BMC 提供带外管理通道，即使操作系统崩溃或 CPU 宕机，管理员仍可通过独立网口远程访问服务器进行诊断和修复。

### 内存子系统
消费级主板使用双通道 UDIMM（Unbuffered DIMM），最多 4 条插槽，单条容量通常 48GB 封顶。服务器主板使用 8~12 通道 RDIMM（Registered DIMM）或 LRDIMM（Load-Reduced DIMM）——RDIMM 在地址/命令线上加了寄存器，降低主板走线的电气负载，使单通道可以驱动更高容量和更多 rank 的内存条；LRDIMM 进一步在数据线上加缓冲，可将单条容量推到 256GB。

Genoa 平台的 12 通道 DDR5-4800 理论带宽约 460 GB/s，是消费级双通道 DDR5-6400（约 102 GB/s）的 4.5 倍。带宽差异在大规模矩阵运算、内存数据库、CFD 仿真等流式工作负载上尤为关键——这些场景下 CPU 的实际吞吐量往往受限于内存带宽而非算力。

### 电源与散热
服务器主板的供电设计目标是持续满载稳定性，而非峰值超频能力。CPU VRM 通常采用多相数字供电（如 SP5 平台的 20+ 相），每相配备服务器级 DrMOS 和钽电容，确保在 350-500W TDP 持续负载下纹波控制在极低水平。散热方案为被动式——服务器主板本身不带风扇，依赖机箱内的暴力风扇（1U 服务器常用 40mm 万转风扇）产生的风压强制散热。这要求主板上的散热片设计为低矮的鳍片形态，配合特定风道方向。

## EPYC 处理器命名规则
AMD EPYC 使用四位数字 + 可选后缀的命名体系，以 7601 为例拆解：

`7` — 产品系列，7 代表 7000 系列（SP3 平台），9 代表 9000 系列（SP5 平台），8 代表 8000 系列（SP6 边缘计算平台）。

`6` — 核心数档位，数字越大核心越多但并不精确等于数量，只表示所属的核心数区间。7001 (Naples) 代际的实际对应关系为：2 = 8 核 (7251/7261)、3 = 16 核 (7301/7351)、4 = 24 核 (7401/7451)、5 = 32 核 (7501/7551)、6 = 32 核旗舰款 (7601)。到了 7002/7003 代际核心数整体上移，6 对应到 48~64 核区间。所以解读核心数档位时必须结合第四位的代际数字一起看。

`0` — 性能/定位细分，在同一核心数档位内区分高低配。第三位数字越大概率上代表更高的主频或更大的缓存配置。例如 7601 的核心数档位为 6（32 核），定位细分 0 意味着它是 32 核中的标准性能款；如果是 6 则属于同核心数下的高频/高性能款。

`1` — 架构代际，1 = Zen 1 (Naples)、2 = Zen 2 (Rome)、3 = Zen 3 (Milan)、4 = Zen 4 (Genoa/Bergamo/Siena)、5 = Zen 5 (Turin)。这是命名中最重要的单字符——它直接决定了 IPC、能效比和指令集支持（如 AVX-512 从 Zen 4 开始引入）。

后缀用于标记特殊能力：
+ `P` 后缀（如 7551P）：仅支持单路（1P），不能用于双路服务器，但通常价格更低。
+ `F` 后缀（如 9374F）：高频优化款，面向单线程延迟敏感型负载（如高频交易、EDA 仿真）。
+ `X` 后缀（如 7773X）：搭载 3D V-Cache，在 CCD 上堆叠额外 L3 缓存，适用于计算流体力学、EDA 等对缓存容量极度敏感的工作负载。Milan-X 的 3D V-Cache 将 L3 从 256 MB 提升至 768 MB。
+ `S` 后缀（如 9754S）：面向云原生/高密部署，使用 Zen 4c 密集核心，在相同功耗下塞入更多核心。

以 7601 为例验证这条规则：7 系列 + 6=32 核 + 0=标准性能定位 + 1=Zen 1 架构 + 无后缀=支持双路。7601 实际参数为 32C/64T、基频 2.2 GHz、TDP 180W、8 通道 DDR4-2666、128 条 PCIe 3.0——完全符合命名规则给出的预期。

## 企业级主板平台（Socket）
+ SP3 (LGA 4094)：AMD 的第一代可扩展服务器插座，覆盖 Zen 1 到 Zen 3 三代架构（2017-2022）。这个平台最突出的特点是三代 CPU 共享同一插座——从 Naples 到 Milan，一块 SP3 主板可以通过 BIOS 更新跨越三代 CPU。8 通道 DDR4 和 128 条 PCIe 4.0 的规格在推出时远超同代 Intel Xeon 平台。SP3 是目前数据中心中保有量最大的 EPYC 平台。
+ SP5 (LGA 6096)：为 Zen 4/Zen 5 设计的旗舰平台（2022-），插座物理尺寸 76×80 mm，是三个平台中最大的一档。12 通道 DDR5（Genoa 4800 MT/s，Turin 6000 MT/s）和 160 条 PCIe 5.0 通道使其成为 HPC 和 AI 训练场景的主力。支持双路互联和 CXL 1.1+（64 条 lane 可配置为 CXL），TDP 上限在 Turin 代达到 500W。Genoa 到 Turin 保持插座兼容，为已有 SP5 基础设施提供了原地升级路径。
+ SP6 (LGA 4844)：面向边缘计算和电信场景的低功耗平台（2023-）。插座物理尺寸与 SP3 相同（58.5×75.4 mm），但引脚数增至 4844。仅支持单路，6 通道 DDR5 和 96 条 PCIe 5.0 的规格仅为 SP5 的一半，对应地 TDP 可低至 70W。SP6 的关键价值在于能在电信机架、零售边缘节点等空间和散热受限的环境中提供 x86 服务器级别的可靠性和 ECC 内存支持，替代大量散落的 Xeon-D 设备。

SP3、SP5、SP6 三个插座之间物理和电气上互不兼容。换个说法，每换一代平台就需要更换主板和散热器。SP6 和 SP3 虽然尺寸相同（散热器孔位兼容），但引脚定义不同，不能混插 CPU。

## 单路与双路拓扑
服务器主板按 CPU 数量分为单路（1P）和双路（2P）两种拓扑。单路主板只焊一个 CPU 插座，所有 PCIe 通道和内存通道由一颗 CPU 独占，适合中小型负载和成本敏感场景。双路主板焊接两个插座，两颗 CPU 通过专用的高速互联总线（AMD 称为 Infinity Fabric）连接，共享内存地址空间但各自维护本地内存控制器。

双路拓扑的关键工程考量是 NUMA（Non-Uniform Memory Access）。CPU 0 访问自己本地的 8 通道内存延迟最低（本地访问），访问 CPU 1 的内存需要跨越 Infinity Fabric（远程访问），延迟增加约 50-100%。对于内存带宽敏感的应用（如数据库、虚拟机），操作系统需要通过 NUMA 亲和性调度，将进程绑定到某个 NUMA 节点，避免频繁的跨节点内存访问。Linux 的 `numactl` 和 `libnuma` 提供了这种绑定的接口。

是否选择双路取决于工作负载类型。单颗 64 核 EPYC 的总吞吐量通常优于两颗 32 核 EPYC——因为单路下所有核心共享同一内存控制器和 PCIe 根复合体，没有跨节点互联开销。双路的价值在于内存容量翻倍（2×8=16 通道）和 PCIe 通道翻倍（2×128=256 条），适合需要超大内存或大量扩展卡的场景。现代趋势是"单路大核"——用一颗 96 核甚至 128 核的 EPYC 替代双路 48 核配置，既省去 NUMA 调优的麻烦，又降低了主板和互联的复杂度。

## BMC 带外管理
BMC（Baseboard Management Controller）是服务器主板上独立于主 CPU 的一个低功耗管理芯片，主流方案是 ASPEED 的 AST2500/AST2600 系列。BMC 有自己独立的 ARM 处理器、固件（如 OpenBMC、AMI MegaRAC）和网卡，通过 I2C、SMBus、PCIe 与主板各组件连接，即使主系统断电（只要插着电源线）BMC 仍然在线。

BMC 提供的核心能力包括：带外管理（通过专用网口或 NC-SI 共享主网口），管理员无需经过操作系统即可远程开关机、查看传感器读数、查看事件日志；远程 KVM（BMC 内置视频捕获芯片，模拟键鼠输入，实现类似物理在场的 BIOS/UEFI 操作）；虚拟介质（把远程的 ISO/IMG 挂载为服务器的虚拟光驱/U 盘，实现远程装机）；Serial-over-LAN（远程串口，用于内核调试）。

BMC 对外的标准协议是 IPMI（Intelligent Platform Management Interface）和现代的 Redfish（基于 REST API 的 DMTF 标准）。IPMI 已逐渐被 Redfish 取代，但 BMC 仍然兼容两者。运维实践中，BMC 是服务器远程运维的核心入口——批量部署操作系统、硬件故障告警、远程故障恢复都依赖 BMC。这也是消费级主板和服务器主板最显著的区别之一：消费级主板没有 BMC，远程管理只能依赖操作系统层面的工具（如 SSH、远程桌面）。

## 厂商介绍
**国际大厂**

+ **Dell PowerEdge** — 服务器市场份额全球第一。PowerEdge 覆盖从单路入门到四路旗舰，R 开头为机架式（R7625 = 双路 EPYC 2U）、T 为塔式、XE 为边缘/工业。iDRAC（Dell 的 BMC 实现）功能成熟，从虚拟介质挂载、远程固件更新到自动故障告警和 SupportAssist 主动报修，开箱即用。主板全部定制设计不单独出售。
+ **HPE ProLiant** — 惠普企业级产品线，DL 为机架式、ML 为塔式。iLO（Integrated Lights-Out）是最早的 BMC 实现之一，iLO 6 已支持 Redfish API 和 HTML5 远程控制台。核心优势是全球服务网络——4 小时上门备件更换对金融和电信客户有实际价值。
+ **Lenovo ThinkSystem** — 收购 IBM x86 服务器业务（System x）后建立的品牌。SR 系列机架式在设计密度上有优势——1U 机箱内支持 32 条 DIMM 和 2 块 GPU 的设计在同类产品中少见。BMC 称为 XClarity Controller，与企业级 XClarity Administrator 管理平台集成。
+ **Supermicro** 超微— 硅谷公司，以提供裸主板和准系统为核心差异，是自建服务器的首选渠道。优势在于产品迭代快（新平台发布后数周内出板）、SKU 丰富（同代芯片组有数十款不同板型）、价格低于品牌服务器。劣势是 BMC 固件交互不如一线大厂，管理功能需要额外的运维能力。
+ **Gigabyte / ASRock Rack** 技嘉/华擎— 台湾两大主板厂的服务器部门。Gigabyte R/M 系列面向 GPU 服务器和小型 HPC，ASRock Rack 性价比突出，ROMED8 系列在自建 NAS 中大量使用。BMC 是 ASPEED AST2500/2600 公版方案加少量定制 UI。
+ **TYAN** 泰安— 台湾老牌服务器主板厂商，1989 年成立，现为神达（MiTAC）旗下品牌。TYAN 专注于服务器主板和准系统的 ODM/OEM 和零售业务，产品线覆盖单路到四路 EPYC 和 Xeon 平台。与 Supermicro 类似，TYAN 也单卖裸主板，在自建服务器和小型数据中心中有一定市场份额。典型产品如 Transport CX 系列 GPU 服务器准系统。

**国产寨板厂商**

在主流品牌之外，国内市场还存在一批使用回收或库存服务器芯片组改造的消费级主板，俗称"寨板"。这类产品的主要目标用户是预算极度有限的 DIY 用户和 E5 洋垃圾爱好者。

这类主板的选购逻辑与主流消费级主板不同：不追求扩展性（通常只有 1-2 个 M.2 和基本 I/O）、不追求高频（E5 系受限于 CPU 内存控制器，DDR4 上限通常 2400-2666 MHz）、不做超频——核心卖点是"用最低成本获得多核计算能力"。一块 X99 主板 + 一颗 18 核 E5-2696 v3（二手约 150 元）的组合，总成本约 600 元即可获得 36 线程的算力，适合预算有限的 CPU 密集型场景（如代码编译、3D 渲染、本地 CI/CD）。

+ **华南金牌（Huananzhi）** — 国产寨板的代表品牌，以 X99、X79 芯片组主板起家。核心模式是回收拆机服务器芯片组和 CPU 插座，重新 Layout 为兼容消费级 ATX 机箱和散热器的主板。代表性产品包括 X99-TF（支持 E5 v3/v4 处理器、DDR4 四通道、NVMe M.2、价格通常在 400-600 元区间）。华南金牌的 BIOS 基于公版 AMI Aptio 修改，相比一线厂商的 UEFI 界面简陋，但基本功能（超频、风扇控制、XMP）可用。品控和售后是主要风险——不同批次的用料和稳定性差异大，建议从官方旗舰店购买以获得基本售后保障。
+ **精粤（Jginyue/Jingyue）** — 与华南金牌定位相近，以 X99、C612 芯片组主板和 B760、B650 等兼容一线 CPU 的低价"平替"主板为主。精粤的亮点是较早布局了 AMD 消费级平台（如 B550、B650 的平替方案），价格显著低于一线品牌同芯片组产品。缺点是 BIOS 更新频率低、QVL 覆盖有限、内存兼容性问题较多——特别是对高频率 DDR5 和冷门品牌内存的支持不如一线稳定。

### 个人自建推荐
个人搭建本地服务器与数据中心选型逻辑不同——重点不在品牌整机的售后服务，而在能否买到裸主板、BMC 是否好用、能否装入消费级机箱。

**首选 Supermicro**。它是自建服务器最成熟的选择——裸主板和准系统均可单独购买，H11（SP3/Rome）、H12（SP3/Milan）、H13（SP5/Genoa）系列覆盖了从 ATX 到专有板型的完整产品线。一块二手的 H11SSL-i（SP3, ATX, 单路 Rome/Milan）价格在 1500-2500 元，装入标准 ATX 机箱配合普通 ATX 电源即可运行。Supermicro 的 BIOS 更新和 IPMI 文档在社区中有大量资料，BMC 固件虽然界面粗糙但功能完整。

**次选 ASRock Rack**。性价比更高的零售服务器主板。ROMED8-2T（SP3, ATX, 单路 Rome/Milan, 双 10G 网口）的二手价格常在 1200-1800 元区间，板型标准、兼容标准机箱。ASRock Rack 的 BMC 基于 ASPEED 公版方案，功能与 Supermicro 基本一致。劣势是固件更新频率不如超微，对小众操作系统的兼容性验证较少。

**TYAN** 适合对特定板型有需求的用户——例如需要 7 条 PCIe x16 或更多 SlimSAS 接口。TYAN 的零售渠道不如前两者广，二手市场流动性也较低，但对特殊配置（如 8 GPU 全 x16 带宽）往往是唯一选择。

**不推荐 Dell、HPE、Lenovo 整机**给个人用户。这些品牌的二手服务器（如 R730、DL380 Gen9）虽然极便宜（几百到一千元），但面临三个致命问题：一是噪音——1U/2U 服务器的 40mm 风扇在自检后会全速运转，住宅环境根本无法忍受，除非有独立机房或车库；二是功耗——双路 Xeon v3/v4 平台即使空闲也 150W+；三是固件锁——品牌服务器的 BIOS/BMC 固件更新通常需要有效维保合同才能下载，二手设备没有合法固件来源。如果确实需要品牌服务器，最低配置是 Dell T 系列或 HPE ML 系列的塔式机型——使用 120mm 风扇，噪音可接受，且通常与标准 ATX 电源兼容。

**国内市场对个人用户而言二手拆机是主要渠道**。选择逻辑与国际品牌完全不同,大部分国内服务器厂商只做整机或 ODM 定制，不单卖主板。个人搭建场景下，浪潮、超聚变、联想（ThinkSystem）的拆机主板在二手市场上流通量最大，价格远低于同规格国际品牌。Supermicro 和 ASRock Rack 的国产新板直接从淘宝/京东授权店购买即可，不需要考虑二手。

## 服务器主板板型
标准的 EEB（12×13 英寸）和 E-ATX 是零售市场最常见的尺寸，能装入多数 E-ATX 兼容的全塔机箱。品牌服务器大量使用定制板型，走线和散热都针对特定机箱优化，几乎没有跨品牌互换的可能。机架式服务器按 U（1U = 1.75 英寸）划分：1U/2U 追求密度，主板为超长窄形（深 650-800mm），CPU、内存、PCIe 全部水平排列；4U+ 可安装全高扩展卡和大型散热，主板接近 EEB 尺寸。
