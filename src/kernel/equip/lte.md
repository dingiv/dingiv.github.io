# 移动网络模块

4G


5G


研究 4G 模块时会发现，它们插在普通的转接板上几乎“一插就准”，原因如下：

总线协议简单：绝大多数 4G 模块走的是 USB 2.0/3.0 协议，而不是 RM520 这种复杂的 PCIe + MHI 协议。USB 协议自带完善的容错机制和热插拔管理，不会像 PCIe 那样对时序（Reset）要求到毫秒级。

功耗控制：4G 模块峰值电流一般在 1A 左右，普通的 PCIe 转接板供电绰绰有余。而 5G 模块在搜网时巨大的瞬时功耗会直接拉跨劣质转接板的电压。

驱动成熟度：4G 模块的 option 或 qmi_wwan 驱动在 Linux 内核里已经躺了十几年了，稳定性极高。
